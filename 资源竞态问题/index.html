资源竞态问题解决方案演示

方案一：Promise 方案（队列机制）

使用 Promise 和 async/await 实现队列机制。所有请求并行发送，但结果严格按照点击顺序依次处理。 即使先发起的请求响应较慢，也会等待其完成后再处理后续请求。
处理结果（按点击顺序）：
暂无数据，请点击上方按钮
[系统]等待操作...
💡 测试提示：快速连续点击多个按钮（例如：3→1→5→2），观察结果严格按照点击顺序依次出现
// 使用示例
const solution = new PromiseSolution();

// 注册自定义更新函数
solution.registerUpdateCallback((data, index, waitTime) => {
  // 在这里实现自定义的 UI 更新逻辑
  console.log(`按钮 ${data.buttonId} 已处理（序号: ${index}）`);
  updateUI(data, index);
});

方案二：非 Promise 方案（回调队列）

使用传统回调函数实现队列机制。通过轮询方式检查数据是否返回，确保按顺序处理。 不依赖 Promise，适合需要兼容旧代码的场景。
处理结果（按点击顺序）：
暂无数据，请点击上方按钮
[系统]等待操作...
💡 测试提示：与 Promise 方案相同，点击多个按钮观察顺序处理效果
// 使用示例
const solution = new CallbackSolution();

// 注册自定义更新函数
solution.registerUpdateCallback((data, index, waitTime) => {
  // 在这里实现自定义的 UI 更新逻辑
  console.log(`按钮 ${data.buttonId} 已处理（序号: ${index}）`);
  updateUI(data, index);
});
