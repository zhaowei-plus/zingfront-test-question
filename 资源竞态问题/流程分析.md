# 资源竞态问题解决方案流程分析

## 问题描述

用户界面上有多个按钮，每个按钮点击会触发一个异步请求（例如从服务器获取数据），用户可以以任意顺序快速点击这些按钮。

**核心要求：**
- 避开竞态问题，结果返回的顺序影响 UI 渲染结果
- 支持 N 个按钮的并发处理
- 请求应该**并行发送**，结果应该**串行返回**
- 每次返回结果，用户都能看到 UI 的变化效果，像动画一样
- 提供两种解决方案：Promise 方案和非 Promise 方案

---

## 方案一：Promise 方案（队列机制）

### 核心思想
使用 Promise 和 async/await 实现队列机制，通过 `await` 等待每个任务完成，确保按顺序处理结果。

### 数据结构

```javascript
class PromiseSolution {
    queue = [];           // 任务队列
    isProcessing = false; // 是否正在处理队列
    updateCallback = null; // 自定义更新函数
    nextExpectedIndex = 1; // 下一个期望处理的序号
}
```

### 处理流程

#### 步骤1：初始化
```javascript
const solution = new PromiseSolution();
solution.registerUpdateCallback((data, index, waitTime) => {
    // 自定义UI更新逻辑
});
```

#### 步骤2：发起请求（并行发送）
当用户点击按钮时：

```
用户点击按钮 → requestData(buttonId)
            ↓
        创建任务对象
        {
            id: 时间戳,
            buttonId: 按钮ID,
            promise: mockAsyncRequest(buttonId), // Promise 对象
            startTime: 发起时间
        }
            ↓
        加入队列: queue.push(task)
            ↓
        如果队列未处理 → processQueue()
```

**关键点：**
- 每个请求立即发起，**并行发送**
- 所有请求的 Promise 同时在执行
- 不会等待上一个请求完成再发起下一个

#### 步骤3：处理队列（串行处理）

```
processQueue()
    ↓
检查队列状态
    ↓
if (queue.length === 0 || isProcessing) → 返回
    ↓
isProcessing = true
    ↓
while (queue.length > 0) {
    const task = queue.shift(); // 取出第一个任务
    ↓
    await task.promise; // 等待当前任务完成（关键！）
    ↓
    调用 updateCallback(data, index, waitTime); // 处理结果
    ↓
    nextExpectedIndex++; // 序号递增
}
    ↓
isProcessing = false
```

**核心机制：**
```javascript
// 使用 await 等待当前任务完成
const data = await task.promise;
```

- **await 确保阻塞**：即使后面的请求先返回数据，也会等待前面的任务完成
- **按队列顺序处理**：严格按照 `queue.shift()` 的顺序处理
- **串行化输出**：结果按点击顺序依次输出

### 时间线示例

假设用户依次点击按钮：3 → 1 → 5 → 2

```
时间轴：
t=0ms:    点击按钮3 → task3加入队列 → processQueue开始
t=10ms:   点击按钮1 → task1加入队列（并行发送）
t=20ms:   点击按钮5 → task5加入队列（并行发送）
t=30ms:   点击按钮2 → task2加入队列（并行发送）

t=100ms:  task1 返回（最快），但被 await 阻塞，等待task3
t=1500ms: task3 返回 → 处理task3 → 输出[1]
t=1800ms: task5 返回 → 处理task1 → 输出[2]
t=2200ms: task2 返回 → 处理task5 → 输出[3]
t=2500ms: 处理task2 → 输出[4]

最终输出顺序：3 → 1 → 5 → 2（与点击顺序一致！）
```

### 优缺点

**优点：**
- 代码简洁，使用现代 async/await 语法
- 机制清晰，易于理解和维护
- 自动处理异步等待

**缺点：**
- 依赖 Promise，不兼容旧浏览器
- 如果第一个请求很慢，会阻塞所有后续处理

## 方案二：非 Promise 方案（回调队列）

### 核心思想
使用传统回调函数和轮询机制，通过定期检查队列中的任务数据是否返回，确保按顺序处理。

### 数据结构

```javascript
class CallbackSolution {
    queue = [];           // 任务队列
    isProcessing = false; // 是否正在处理队列
    updateCallback = null; // 自定义更新函数
    nextExpectedIndex = 1; // 下一个期望处理的序号
}
```

### 处理流程

#### 步骤1：初始化
```javascript
const solution = new CallbackSolution();
solution.registerUpdateCallback((data, index, waitTime) => {
    // 自定义UI更新逻辑
});
```

#### 步骤2：发起请求（并行发送）
当用户点击按钮时：

```
用户点击按钮 → requestData(buttonId)
            ↓
        创建任务对象
        {
            id: 时间戳,
            buttonId: 按钮ID,
            startTime: 发起时间,
            data: null // 初始为空
        }
            ↓
        加入队列: queue.push(task)
            ↓
        发起异步请求（回调方式）
        mockAsyncRequestCallback(buttonId, (error, data) => {
            if (!error) {
                // 将结果存入对应的任务
                task.data = data;
            }
        })
            ↓
        如果队列未处理 → processQueue()
```

#### 步骤3：处理队列（轮询方式）

```
processQueue()
    ↓
检查队列状态
    ↓
if (queue.length === 0 || isProcessing) → 返回
    ↓
isProcessing = true
    ↓
processNext() 开始轮询
```

#### 步骤4：轮询处理

```
processNext()
    ↓
if (queue.length === 0) → isProcessing = false; 返回
    ↓
const task = queue[0]; // 查看第一个任务
    ↓
if (task.data !== null) {
    // 数据已返回 → 立即处理
    ↓
    调用 updateCallback(data, index, waitTime);
    ↓
    nextExpectedIndex++;
    ↓
    queue.shift(); // 移除已处理的任务
    ↓
    processNext(); // 递归处理下一个
} else {
    // 数据还未返回 → 等待100ms后再次检查
    ↓
    setTimeout(() => {
        processNext(); // 轮询
    }, 100);
}
```

**核心机制：**
```javascript
if (task.data) {
    // 数据已返回，处理
    handleData(task);
} else {
    // 数据未返回，100ms后重试
    setTimeout(processNext, 100);
}
```

- **轮询检查**：每100ms检查一次队列首个任务的数据是否返回
- **阻塞机制**：首个任务未返回前，不会处理后续任务
- **串行化输出**：结果按队列顺序依次输出

### 时间线示例

假设用户依次点击按钮：3 → 1 → 5 → 2

```
时间轴：
t=0ms:    点击按钮3 → task3加入队列 → processQueue开始轮询
          轮询1：task3.data = null → 等待100ms
          
t=10ms:   点击按钮1 → task1加入队列（并行发送）
t=20ms:   点击按钮5 → task5加入队列（并行发送）
t=30ms:   点击按钮2 → task2加入队列（并行发送）

t=100ms:  task1 返回 → task1.data = {buttonId:1, ...}
          轮询2：task3.data = null → 等待100ms
          
t=200ms:  轮询3：task3.data = null → 等待100ms
t=300ms:  轮询4：task3.data = null → 等待100ms
...
t=1500ms: task3 返回 → task3.data = {buttonId:3, ...}
          轮询16：task3.data !== null → 处理task3 → 输出[1]
                   task3从队列移除
                   task1成为首个任务 → 处理task1 → 输出[2]
                   task1从队列移除
                   task5成为首个任务
                   
t=1600ms: 轮询17：task5.data = null → 等待100ms
t=1700ms: task5 返回 → task5.data = {buttonId:5, ...}
          轮询18：task5.data !== null → 处理task5 → 输出[3]
                   task2成为首个任务 → task2.data已存在
                   处理task2 → 输出[4]

最终输出顺序：3 → 1 → 5 → 2（与点击顺序一致！）
```

### 优缺点

**优点：**
- 不依赖 Promise，兼容性更好
- 机制直观，容易理解
- 适合旧代码迁移

**缺点：**
- 需要轮询，效率较低（每100ms检查一次）
- 代码相对复杂
- 可能产生额外的CPU开销

## 核心要点总结

### 1. 请求并行发送
- 两个方案都立即发起异步请求
- 不等待上一个请求完成
- 所有请求同时在进行

### 2. 结果串行处理
- 严格按照加入队列的顺序处理
- 即使后面的请求先返回，也会等待前面的任务完成

### 3. 避开竞态问题
- 通过队列机制保证顺序
- Promise 方案使用 await 阻塞
- 回调方案使用轮询检查

### 4. 自定义更新函数
- 用户可以注册任意 UI 更新逻辑
- 完全解耦核心逻辑和 UI 展示
- 支持灵活的扩展

### 5. 实时反馈
- 每个结果返回时立即调用更新函数
- 用户能看到"像动画一样"的连续效果
- 支持逐个处理的视觉反馈

## 使用建议

### 推荐使用 Promise 方案，如果：
- 项目目标浏览器支持 Promise
- 追求代码简洁性和执行效率
- 需要现代化的异步处理

### 推荐使用回调方案，如果：
- 需要兼容旧浏览器
- 正在维护使用传统回调的旧代码
- 不依赖 Promise 特性

## 关键代码片段对比

### Promise 方案核心代码
```javascript
async processQueue() {
    while (this.queue.length > 0) {
        const task = this.queue.shift();
        const data = await task.promise; // 关键：阻塞等待
        this.updateCallback(data, this.nextExpectedIndex++, waitTime);
    }
}
```

### 回调方案核心代码
```javascript
processNext() {
    const task = this.queue[0];
    if (task.data) {
        this.updateCallback(task.data, this.nextExpectedIndex++, waitTime);
        this.queue.shift();
        this.processNext(); // 递归处理下一个
    } else {
        setTimeout(() => this.processNext(), 100); // 轮询
    }
}
```
